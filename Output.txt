Q3. Trigger to automatically update stock when order is placed

Code:-
CREATE TRIGGER UpdateStockAfterOrder
AFTER INSERT ON OrderDetails
FOR EACH ROW
UPDATE Products
SET Stock = Stock - NEW.Quantity
WHERE ProductID = NEW.ProductID;

Core Logic
 - Fetch current stock.
 - Compare it with ordered quantity.
 - If insufficient → throw error.
 - If sufficient → subtract quantity.

Key Concepts Used
BEFORE INSERT
 - NEW.column_name
 - SIGNAL for custom error
 - Transactional integrity

What It Prevents
 - Overselling
 - Negative stock
 - Application-level validation failures

Business Value
 - Prevents overselling
 - Avoids order cancellations due to stock errors
 - Maintains real-time inventory accuracy

Protects brand trust

Q12 — .  How can you rank customers based on their total spending using a window function?

code:-
SELECT 
    c.CustomerID,
    c.Name,
    SUM(o.TotalAmount) AS TotalSpent,
    RANK() OVER (ORDER BY SUM(o.TotalAmount) DESC) AS SpendingRank
FROM Customers c
JOIN Orders o 
    ON c.CustomerID = o.CustomerID
GROUP BY c.CustomerID, c.Name;


Purpose:-
To rank customers by total spending.


Key Concept: Window Function
RANK() OVER (ORDER BY SUM(o.TotalAmount) DESC)

What Happens
 - Calculate total spending per customer
 - Apply ranking over sorted totals
 - Customers with equal spending get same rank
Rank may skip numbers (because RANK() allows ties)

Example
Customer	TotalSpent	Rank
A	            2000	1
B	            1500	2
C	            1500	2
D	            1000	4
Why Use Window Functions?
  - Keeps detail rows
  - More powerful than GROUP BY alone

Business Value
Reward top spenders (VIP programs)
Offer targeted promotions
Prevent churn among high-value users

Q9 — What is the average rating and total reviews for each product?
code :-
SELECT 
    p.ProductID,
    p.ProductName,
    AVG(pr.Rating) AS AverageRating,
    COUNT(pr.ReviewID) AS TotalReviews
FROM Products p
LEFT JOIN ProductReviews pr 
    ON p.ProductID = pr.ProductID
GROUP BY p.ProductID, p.ProductName;


To analyze product performance.
Query Logic
 - LEFT JOIN ensures all products appear (even with no reviews)
 - AVG(Rating) calculates average satisfaction
 - COUNT(ReviewID) counts number of reviews
 - GROUP BY ProductID

Why LEFT JOIN?
Without it, products with zero reviews would not appear.

Business Insight
This query helps:
 - Identify top-rated products
 - Detect poorly rated items
 - Measure customer engagement




Q1 — Average Prices by Category (Filtered > $100)

code:-
SELECT
    Category,
    AVG(Price) AS AveragePrice
FROM Products
GROUP BY Category
HAVING AVG(Price) > 100;

What the Query Does
 - It calculates the average product price per category, then filters out categories whose average price is ≤ 100.

Why Use HAVING Instead of WHERE?
WHERE filters rows before grouping.
HAVING filters results after aggregation.

Logical Execution Flow
Group all products by Category
Compute average price per group
Remove groups where average ≤ 100
Return remaining categories

Business Meaning
This identifies:
Premium categories
High-value product segments
Strategic pricing tiers
